实现一个自己的图床程序

发表于 2024-09-11 | 分类于 编程 ， Java | | 浏览 次
字数统计: 5.6k 字 | 阅读时长 ≈ 分钟 27
平时在博客中，经常使用gitee来作为自己的图床，原来都是用PicGo作为图床软件来上传图片并返回URL，由于最近为了方便自己写博客，搭建了个管理系统，集成了markdown写法。然后其中的上传图片接口直接把上传图片到了gitee，并返回url。然后闲来无事想到了PicGo这个软件，便想着自己来实现一款属于自己的图片上传软件

功能思考

上传到gitee首先需要Token，仓库，用户名以及所属分支，所以添加了一个配置页面，用来保存配置信息，不用每次输入配置信息，预期的是把软件打包成exe随便哪里都能使用，所以为了保存配置信息，这样看来，SQLite是最适合的数据库。界面用JavaFX来编写。

SQLite是一个进程内的库，实现了自给自足的、无服务器的、零配置的、事务性的 SQL 数据库引擎。它是一个零配置的数据库，这意味着与其他数据库不一样，您不需要在系统中配置。
就像其他数据库，SQLite 引擎不是一个独立的进程，可以按应用程序需求进行静态或动态连接。SQLite 直接访问其存储文件
所有功能：

  • 配置Token，仓库，用户名以及所属分支信息
  • 点击选择图片上传并返回url复制到剪切板
  • 剪切板复制了图片直接点击剪切板上传并复制url到剪切板
  • 图片回显
